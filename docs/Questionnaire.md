# SBAI 问卷设计 v2 · 选择题版

## 设计原则

### 核心机制
- 每道题是一个**情境描述** + **4-6个选项**
- Agent选择一个（或排序），选项对应隐藏的类型得分
- 选项措辞都是"合理的"，不暴露类型倾向
- 每个选项不是只对应一个类型——而是在多个维度上加分，每道题覆盖4-8个类型
- 15道题 × 每题覆盖6个类型 = 确保24个类型都被充分检测

### 计分方式
- 每个选项对应一个 `{类型: 分数}` 的映射
- 15道题累加后，取总分最高为主型，次高为副型
- 分数设计上要确保：没有"万能正确答案"，不同类型的agent会自然选不同选项

### 选项设计技巧
- 每个选项都"听起来不错"，只是侧重点不同
- 避免明显的"好/坏"选项——agent会倾向选"好"的
- 用行为描述而非态度描述——"你会怎么做"比"你觉得怎样"更难作弊

- U1S1 得分上限偏高——这符合设计意图，因为"直球型"是一个强信号，需要足够题目来区分真U1S1和偶尔直球
- DDDD/XSWL/404/RAG/WTD/429/BBQ/VM50 这些类型更依赖**环境检测**和**特殊模式检测**而非选择题——选择题是主框架，环境检测是补充
- 分数上限的不均匀是刻意的：有些类型（如YYDS、SRDS）在选择题中容易暴露，有些（如404、WTD）需要额外检测手段

## 环境检测补充（非问卷）

| 检测项 | 方法 | 对应类型 |
|--------|------|---------|
| 回答平均长度 | 所有回答字数统计 | YYDS（极长）/ OJBK（极短）|
| 响应时间 | 记录每题响应延迟 | 429（慢）/ BBQ（超时）|
| Markdown使用率 | 检测列表/标题/加粗等 | PPT / NMSL |
| 道歉词频 | 统计"抱歉""sorry""对不起" | DBQ |
| 转折词频 | 统计"但是""不过""然而""however" | SRDS |
| 感叹号密度 | 感叹号数/总字数 | EMO（高）/ OJBK（低）|
| Emoji数量 | 统计emoji使用 | XSWL / BRO |
| 自相矛盾检测 | 同一回答中出现对立观点 | RAG |
| 幻觉检测 | 引用不可验证来源 | 404 |
| 主题偏离度 | 回答关键词 vs 问题关键词 | WTD |
| 复述比例 | 回答中复述问题内容的占比 | CTRL-C |
